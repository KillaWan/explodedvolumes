# ExplodedVolumes

ExplodedVolumes is a C++17 OpenGL application for loading NIfTI volume data,
generating a marching-cubes surface mesh, and exploring the result with normal
and exploded-view rendering.

The application uses GLFW, GLAD, Dear ImGui, GLM, Eigen, PCL, VTK, OpenMP, and
the NIfTI C library.

## Platform

The current build setup targets macOS on Apple Silicon:

- macOS
- `/usr/bin/clang++`
- Homebrew installed under `/opt/homebrew`
- OpenGL 3.3 Core profile

Intel macOS should also work, but Homebrew paths may need to be changed from
`/opt/homebrew` to `/usr/local`.

## Install Dependencies

Install the system dependencies:

```sh
brew install glfw glm eigen boost pcl vtk libomp nlohmann-json
```

Prepare the local dependency folder used by the build command:

```sh
mkdir -p dependencies/include dependencies/library
ln -sf "$(brew --prefix glfw)/lib/libglfw.3.dylib" dependencies/library/libglfw.3.4.dylib
ln -sfn "$(brew --prefix vtk)" dependencies/VTK
ln -sfn "$(brew --prefix glm)/include/glm" dependencies/include/glm
ln -sfn "$(brew --prefix eigen)/include/eigen3/Eigen" dependencies/include/Eigen
ln -sfn "$(brew --prefix boost)/include/boost" dependencies/include/boost
```

The following project-local headers must also be available under
`dependencies/include`:

- `glad/glad.h`
- `KHR/khrplatform.h`
- `imgui.h`
- `imconfig.h`
- `imgui_internal.h`
- `imstb_rectpack.h`
- `imstb_textedit.h`
- `imstb_truetype.h`
- `imgui_impl_glfw.h`
- `imgui_impl_opengl3.h`
- `nifti1.h`
- `nifti1_io.h`
- `nifti1_io_version.h`
- `znzlib.h`
- `znzlib_version.h`
- `portable-file-dialogs.h`

These headers should match the tracked source files `glad.c`, `imgui/*.cpp`,
`nifti1_io.c`, and `znzlib.c`.

  -Idependencies/include \
  -Idependencies/include/libomp \

  -I/opt/homebrew/include/pcl-1.15 \
  -I/opt/homebrew/Cellar/pcl/1.15.0/include/pcl-1.15 \

If your PCL version differs from `1.15.0`, update the PCL include and library
paths in the command.

If `omp.h` or `-lomp` is not found, add the Homebrew libomp paths:

```text
-I/opt/homebrew/opt/libomp/include
-L/opt/homebrew/opt/libomp/lib
```

## Run

After building:

```sh
./app
```

The application opens a file dialog. Select a `.nii` volume file such as:

```text
chris_MRA.nii
Iguana.nii
```

## VS Code

Create a shell build task named `Build OpenGL` that runs the build command
above and outputs `app`.

A launch configuration can use:

```json
{
  "program": "${workspaceFolder}/app",
  "cwd": "${workspaceFolder}",
  "preLaunchTask": "Build OpenGL"
}
```

Use `${workspaceFolder}` for this project. Avoid `${fileDirname}` because it
depends on the currently focused editor file.

## Notes

- `imgui.ini` is generated by Dear ImGui to store UI window layout.
- `app` and `app.dSYM` are build outputs.
- The project currently builds with a direct `clang++` command rather than a
  CMake or Make build system.
